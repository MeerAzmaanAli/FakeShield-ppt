05
Blockchain Layer
Hardhat · Solidity · Ethereum

Immutable audit.
Tamper-proof records.

I
Immutable Audit Log

Every report submission and agency verdict is hashed on-chain. Cannot be altered retroactively — the chain is the source of truth.

II
Tamper-Proof Records

Smart contracts enforce submission rules. No single party can modify verdicts post-submission. Code is law.

III
Decentralized Trust

No central database holds the truth — the chain is the record, transparent to all authorized parties. No single point of failure or manipulation.

IV
Gov Accountability

Agency actions are permanently logged, creating accountability and enabling dispute resolution. Removes the conflict-of-interest of self-policing.

06
User Workflow
Five-step Process

From submission
to blockchain verdict

01
Submit Profile

User enters a social media profile URL or handle on the public portal. Works across Instagram, Facebook, and X.

02
ML Scoring

Scraper API fetches live account features. Random Forest model returns a fakeness probability and risk label.

03
File Report

User reviews the ML score and submits a formal report with supporting evidence through the portal.

04
Agency Review

A government analyst reviews the case in a secure, role-authenticated dashboard with ML-enriched data.

05
Blockchain Log

The final verdict — Fake / Genuine / Inconclusive — is recorded on-chain. Tamper-proof, permanent, auditable.
